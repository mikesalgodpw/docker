#Remarks for command "gcloud auth configure-docker":
if it's the first time, u run this command,
then u need to input "Y" in the inteactive conosle.


Remarks for command "docker push gcr.io/${PROJECT_ID}/test-tc:v1":
1. If the image is gcp VM "centos-slave", then u need to run the push command 
   on the gcp VM "centos-slave".

2. If run "docker push gcr.io/${PROJECT_ID}/test-tc:v1",
   u may get "An image does not exist locally with the tag: gcr.io/cpb100-188605/test-tc
"
  --> docker tag test-tc gcr.io/${PROJECT_ID}/test-tc:v1
  
  Extra:
  if u do NOT tag, and push directly using "docker push",
  u will get "permission deny". coz dokcer will assume u r going to push to: docker.io

#deployment
